# Проверка компаний ПлюсЗвено

Модуль формирует отчет по компании или ИП по ИНН. Это не витрина для пользователя, а рабочий механизм для интерфейса администратора и представителя ПлюсЗвена.

## Запуск

```powershell
cd C:\Users\t8950\opo-service
py .\company_check\company_check.py --inn 7707083893
```

## Checko API key

The admin UI runs `company_check.ps1`. It reads the Checko key from `CHECKO_API_KEY`.
Do not commit the key into this repository.

For the current PowerShell window:

```powershell
$env:CHECKO_API_KEY = "your_checko_key"
powershell -ExecutionPolicy Bypass -File .\company_check\company_check.ps1 -Inn 7707083893
```

For the current Windows user:

```powershell
[Environment]::SetEnvironmentVariable("CHECKO_API_KEY", "your_checko_key", "User")
```

Open a new PowerShell window after setting the user environment variable.
If Checko changes the endpoint, set `CHECKO_API_BASE`; the default is `https://api.checko.ru/v2/company`.

Если `py` или Python не установлен, нужно поставить Python 3.10+ и повторить запуск.

Выходные файлы пишутся в `company_check/out`:

- `company_<inn>_<date>_<hash>.json` - структурированный отчет для ИА/API.
- `company_<inn>_<date>_<hash>.md` - читабельная выжимка для представителя.

## Статусы источников

- `ok` - источник отработал и вернул данные.
- `error` - автоматический сбор был возможен, но завершился ошибкой.
- `manual_review` - есть публичный источник, но нет стабильного публичного API без токена/капчи/интеграции.
- `configured` - нужен внутренний файл, база, токен или партнерская интеграция.

## Откуда и что собираем

### ФНС: ЕГРЮЛ/ЕГРИП

Автоматический источник. По ИНН собираем:

- юридическое название;
- ОГРН;
- ИНН;
- КПП;
- адрес;
- руководителя;
- статус юрлица;
- дату регистрации;
- тип записи.

Использование: базовая юридическая карточка компании и первичная проверка, что компания существует и не имеет явного признака прекращения деятельности.

### ФНС: ГИР БО

Пока `manual_review`. Собираем:

- выручку;
- прибыль/убыток;
- активы;
- динамику отчетности.

Использование: финансовый контекст, способность компании выполнять платежные обязательства, признаки пустой или проблемной компании.

### Федресурс

Пока `manual_review`. Собираем:

- сообщения о банкротстве;
- ликвидацию/реорганизацию;
- существенные факты;
- залоги и уведомления, если применимо.

Использование: критичные сигналы перед публикацией вакансий и подтверждением компании.

### Картотека арбитражных дел

Пока `manual_review`. Собираем:

- дела в роли ответчика;
- платежные и подрядные споры;
- массовость исков;
- суммы требований.

Использование: риск невыплаты, конфликтность, история подрядных споров.

### ЕИС закупки: РНП

Пока `manual_review`. Собираем:

- наличие в реестре недобросовестных поставщиков;
- основание включения;
- срок нахождения в реестре.

Использование: публичный репутационный сигнал для карточки компании.

### ФССП

Пока `manual_review`. Собираем:

- активные исполнительные производства;
- суммы задолженности;
- характер задолженности.

Использование: сигнал платежной дисциплины.

### НОСТРОЙ/НОПРИЗ: СРО

Пока `manual_review`. Собираем:

- членство СРО;
- уровень ответственности;
- приостановления и исключения.

Использование: проверка допуска компании к строительным работам.

### Официальные контакты

Пока `manual_review`. Собираем:

- официальный сайт;
- официальную почту;
- официальный телефон;
- совпадение контактов с вакансией.

Использование: отправка кода подтверждения компании. Код нельзя отправлять на телефон из вакансии, пока он не подтвержден как официальный канал.

### DreamJob

Пока `manual_review`. Собираем:

- общую оценку работодателя;
- количество отзывов;
- положительные и отрицательные темы;
- жалобы на задержки оплаты;
- жалобы на проживание, дорогу, СИЗ и оформление;
- ответы представителя компании.

Использование: репутационный слой. Эти данные не заменяют юридическую проверку, но помогают увидеть повторяющиеся проблемы глазами работников.

### Яндекс Карты

Пока `manual_review`. Собираем:

- карточки компании по ИНН, названию и адресу;
- рейтинг;
- количество отзывов;
- свежие негативные отзывы;
- жалобы работников и подрядчиков;
- совпадение адреса карточки с юридическим или проектным адресом.

Использование: проверка публичной репутации офиса, филиала или объекта. Клиентские отзывы отделяем от отзывов работников и подрядчиков.

### 2ГИС

Пока `manual_review`. Собираем:

- карточки компании по ИНН, названию и адресу;
- рейтинг;
- количество отзывов;
- свежие негативные отзывы;
- жалобы работников и подрядчиков;
- совпадение адреса карточки с юридическим или проектным адресом.

Использование: дополнительная сверка публичной репутации и контактов. Для автоматизации нужен официальный API/партнерский доступ или разрешенный экспорт.

### Внутренняя история ПлюсЗвена

Источник `configured`. После подключения базы собираем:

- диспуты;
- подтвержденные проекты;
- несоответствия вакансии и договора;
- отзывы специалистов и бригад;
- историю погашения нарушений.

Использование: рейтинг, публичные метки, решение о статусе `Подтверждена`, `Есть сигналы` или `Нарушитель`.

## Статусы компании

- `По открытым данным` - автоматическая проверка не нашла критичных сигналов, но компания еще не подтвердила официальный канал.
- `Подтверждена` - представитель компании назвал код, отправленный на официальный email/телефон из открытых источников. Этот статус будет ставить ИА после ручного подтверждения.
- `Есть сигналы` - найдены предупреждения или неполная проверка требует внимания.
- `Нарушитель` - есть критичные подтвержденные сигналы: банкротство, недействующее юрлицо, подтвержденные долги/споры/манипуляции.

## Что делает представитель после отчета

1. Сверяет ФНС: ИНН, ОГРН, КПП, юридическое название, адрес, руководителя.
2. Находит официальный канал связи на сайте компании, в открытых реестрах, 2ГИС или закупочных документах.
3. Отправляет код подтверждения только на официальный канал.
4. Сверяет вакансию: ставка на руки, НДФЛ, дорога, СИЗ, суточные, проживание, график, тип договора.
5. Запрашивает трудовой договор и сравнивает его с заявленными условиями.
6. Проверяет проект: заказчик/завод, объект, юрлицо завода, руководитель проекта и срок реализации.
7. Сверяет DreamJob, Яндекс Карты и 2ГИС: отделяет клиентские отзывы от отзывов работников и подрядчиков.
8. Фиксирует ручные результаты по судам, Федресурсу, РНП, ФССП и СРО.
